# Integrate OfficeCLI into Beats PM Kit

## Background

[OfficeCLI](https://github.com/iOfficeAI/OfficeCLI) is an open-source, single-binary CLI purpose-built for AI agents to create, read, and edit Word (.docx), Excel (.xlsx), and PowerPoint (.pptx) files — no Office installation required. The kit currently has a barebones `document-exporter` skill that only does Markdown → HTML → PDF via Chrome.

This integration gives every agent in the kit the ability to produce real Office documents — critical for PowerPoint decks, Word reports, and Excel trackers.

## Proposed Changes

### Phase 1: Install OfficeCLI Binary

Run the official Windows installer to place the `officecli` binary on PATH:

```powershell
irm https://raw.githubusercontent.com/iOfficeAI/OfficeCLI/main/install.ps1 | iex
```

This is a one-time setup step (single self-contained binary, .NET runtime embedded).

---

### Phase 2: Create `office-cli` Skill

#### [NEW] `.agent/skills/office-cli/SKILL.md`

A curated skill file (~200 lines) that:

1. **Frontmatter** — name, description, matches kit standard
2. **When to Use** — user requests a PowerPoint, Word doc, Excel sheet, or "export to Office"
3. **Prerequisites** — checks `officecli --version` is installed
4. **Quick Reference** — the most common commands the agent needs:
   - `officecli create <file>` — create blank document
   - `officecli add` — add slides/shapes/paragraphs/cells
   - `officecli set` — modify properties (text, colors, fonts, layout)
   - `officecli view` — inspect structure
   - `officecli batch` — multi-operation atomic writes
   - `officecli watch` — live preview
5. **PowerPoint Workflow** — step-by-step recipe for creating a deck from content
6. **Word Workflow** — step-by-step recipe for creating a report
7. **Excel Workflow** — step-by-step recipe for creating a tracker/dashboard
8. **Common Pitfalls** — curated from upstream (shell quoting, `--prop` syntax, path indexing)
9. **Upstream Reference** — `curl -fsSL https://officecli.ai/SKILL.md` for full reference

#### [NEW] `.agent/skills/office-cli/references/`

A directory containing a cached copy of the upstream SKILL.md for offline use.

---

### Phase 3: Upgrade `document-exporter` Skill

#### [MODIFY] `.agent/skills/document-exporter/SKILL.md`

Add OfficeCLI as a new capability alongside the existing HTML/PDF pipeline:

```markdown
## Capabilities

1. **Markdown → HTML → PDF** (existing) — uses md_to_pdf.py + Chrome
2. **Markdown → DOCX** (NEW) — uses OfficeCLI to create a real Word document
3. **Markdown → PPTX** (NEW) — uses OfficeCLI to create a PowerPoint deck
4. **Markdown → XLSX** (NEW) — uses OfficeCLI to create an Excel workbook

## Format Selection

- User says "PDF" → use existing pipeline
- User says "Word", "DOCX", "document" → delegate to office-cli skill
- User says "PowerPoint", "PPTX", "deck", "presentation" → delegate to office-cli skill
- User says "Excel", "XLSX", "spreadsheet" → delegate to office-cli skill
- User says "export" with no format → ask, default to PPTX for presentations, DOCX for docs
```


> [!IMPORTANT]
> The `documentation-writer` agent gains Office document creation. The `staff-pm` and `cpo` agents can also use it via the `document-exporter` skill which delegates to `office-cli`.
